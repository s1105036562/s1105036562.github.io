npm 介绍及应用

  • npm简单介绍

  • npm 是基于node中的一种工具,当你安装了node 也就又npm
  • npm 是一个包管理器,可以帮助你解决下载第三方包的问题,依赖管理你的包的依赖问题
  • npm 本身是一个网站,所有比较流行的第三方包都在这个网站服务器上,本身就像github一样,存储了很多第三方包

  • 为什么学习npm

  • 安装麻烦
    • 当需要用到这些软件时,还需要一个一个去官网下载,浪费了大量的时间
  • 管理麻烦
    • 当用的第三方软件过多时,不小心操作失误时(删除了),不知道用了哪些第三方软件需要一个一个去找,对于一个项目来讲 有哪些依赖也不很麻烦

  • 使用方法

  • 运行环境

    • 安装node环境
    • node自带了一种工具 npm
    • 去报在控制台输入 npm –version可以看到一个npm版本号就ok

  • npm常用语法

    • npm help 查看npm命令
    • npm init –yes 初始化一个项目 ,会创建一个配置文件 package.json
      • –yes 表示配置的文件全部默认(比较省事)
      • package.json 简单介绍
        • 当我们要在github上 上传项目时 第三方包上传 不但占空间,还浪费时间,但是不上传,当多人协同开发时,页面不能正常运行,也不知道依赖信息,为了解决这个问题,所以 npm init –yes 会初始化一个package.json配置文件 在文件的dependencies 会有 包的依赖信息,当别人开发时,直接down项目,然后 npm install 会自动下载依赖的包
    • 下载包
      • npm install 包名 [–save] (一般用于下载插件比如 jquery 等,也叫本地包)
        • 下载第三方包名到当前目录,下载下来的包会放到node-modules目录中(注意包名必须跟npm官网一致)
        • 可以在包名后面跟上 @版本号 下载指定的版本
        • –save选项是用来告诉npm不仅安装这个包,同时会把依赖信息写入 package.json文件的 dependencies中
        • npm install 当从github上下载的项目 执行该该命令时 会找到项目中的 package.json文件的 dependencies 根据依赖信息下载包,所以我们在项目中装包时一定要写上 –save
      • npm install 包1 包2.. 一次性安装多包
      • npm install –global 包名 该命令用来安装一个全局包,通常是一些工具,而不是jquery这种包 例如 less
        • 安装全局包的时候,在控制台的任意位置执行都可以,全局包就像安装了一个软件一样以后再也不用安装 具体使用查API 安装目录在 C:\Users\Administrator\AppData\Roaming\npm\node_modules
        • 本地包 则要求控制台必须在项目中安装
      • npm root 查看 安装目录
      • npm uninstall 包名 [–save] 卸载本地x项目指定包 –savex选项同时也会把dependenciesx信息也删除
      • npm uninstall –global 包名 卸载全局包
