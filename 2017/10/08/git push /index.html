git push 本地分支和远程分支问题

  • Git push 的完整写法

    • 当我们从GitHub 远程仓库 clone 下来的时候修改完以后可以直接 git push 这只是简写
      • 实际上git push 的完整写法是: git push 远程仓库 本地分支:远程分支

  • 问题所在

    • 当我们第一次把项目down下来时,只有一个 master(主干)分支
    • 当我们在本地创建了一个 feature-foo 分支,然后默认提交时,直接 git push 会提交不上去,需要把 feature-foo 提交到github上 所以正确的写法 是: git push origin feature-foo :feature-foo
      也可以简写为git push origin feature-foo
      ``
    • 也可以直接更改记录的分支 git push –set-upstream orgin feature-foo 下次默认推送分支的就是 feature-foo 在推送feature-foo分支时 可以直接git push
